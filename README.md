# bgenabler.x - X680x0 バックグラウンドタスクイネーブラ

## 概要

Human68kのバックグラウンドタスク機能を再起動せずに有効化、設定変更するためのツールです。Human68k v3.02専用です。

## 特徴

* CONFIG.SYSの書き換えや再起動をせずにバックグラウンドタスク機能を有効化/無効化できます
* CONFIG.SYSの`PROCESS=`行による設定パラメータを起動後に変更できます
* バックグラウンドタスク処理用のタイマーにTimer-Cを使用することで、アプリがTimer-Dを自由に使用できるようになります
  *  ただし、Timer-C使用時はタイムスライス値が10ms単位になります
* バックグラウンドタスク機能使用中でもINTERRUPTスイッチが効きます
* バックグラウンドタスク機能使用中に「COPYキーを押して中断を選択するとクラッシュすることがある」という不具合を回避できるようになります。COPYキーの機能自体を無効にすることもできます
* 「FPUを搭載したX68030がユーザモードでFPU命令を実行するとエラー$000dが発生することがある」という問題を修正します


## 使用方法

**CONFIG.SYS に`PROCESS=`行があると動作しないので、ある場合は削除してバックグラウンド機能無効の状態で実行してください。**
コマンドラインからの実行とCONFIG.SYSによる登録の両方に対応しています。

### コマンドラインからの実行

`bgenabler` コマンドに `-c` または `-d` オプションを付けて実行すると常駐して、バックグラウンドタスク機能が使えるようになります。

バックグラウンド機能有効時もINTERRUPTスイッチが効くようになります。`-c` オプションの場合はTimer-Cを使用するので、IOCS _TIMERDSTがアプリ側で利用できるようになります。

```
A>bgenabler -c        # Timer-Cを使用してバックグラウンドタスク機能を有効化
X680x0 Background Task Enabler version xxxxxxxx
Human68kのタスク切り替え処理にパッチを適用しました
bgnabler が常駐しました
```
```
A>bgenabler -d        # Timer-Dを使用してバックグラウンドタスク機能を有効化
X680x0 Background Task Enabler version xxxxxxxx
Human68kのタスク切り替え処理にパッチを適用しました
bgnabler が常駐しました
```

常駐後に再度 `bgenabler` コマンドを実行すると、バックグラウンドタスク機能の設定を変更できます。コマンドラインオプションによって、`PROCESS=` 行のパラメータで指定できる最大スレッド数や優先度レベル、タイムスライスの設定を再起動せずに変更できます。

```
A>bgenabler -c 32 2 100     # Timer-C使用で最大スレッド数32、優先度レベル2、タイムスライス100msに設定
X680x0 Background Task Enabler version xxxxxxxx
bgnabler の設定を変更しました
```

コマンドラインオプションを付けずに実行すると、現在の設定が表示されます。

```
A>bgenabler
X680x0 Background Task Enabler version xxxxxxxx
常駐状態              : 常駐
バックグラウンドタスク: 有効
BGタスク処理          : Timer-C
最大スレッド数        : 32
メインスレッド優先度  : 2
タイムスライス        : 100ms
COPYキー処理          : 有効
```

`-r`オプションを付けて実行するとバックグラウンドタスク機能を無効化して常駐を解除します。

```
A>bgenabler -r
X680x0 Background Task Enabler version xxxxxxxx
bgenabler の常駐を解除しました
```

### CONFIG.SYSによる登録

CONFIG.SYSの`DEVICE=`行で、デバイスドライバとして登録することができます。
登録時のオプションはコマンドラインでの実行と同様ですが、オプションなしで実行するとバックグラウンドタスク機能が無効化された状態で組み込まれます。

```
DEVICE = bgenabler.x
```

起動後の`bgenabler` コマンド実行では、デバイスドライバとして組み込まれた bgenabler の設定が変更されます。`-r`オプションを付けると、組み込まれたままでバックグラウンドタスク機能が無効化されます。


### コマンドラインオプション

コマンドライン実行、CONFIG.SYS登録の両方で以下のオプションが使用できます。

* `-c`: バックグラウンドタスクの切り替え処理にTimer-Cを使用します
  * Timer-Dの設定を行うIOCS _TIMERDSTがアプリ側で自由に利用できるようになります
  * Timer-C使用時はバックグラウンドタスク機能のタイムスライス値(`PROCESS=`の第3パラメータ)が10ms単位に切り上げられます
  * DOS _TIME_PRで取得できるカウンタ値やDOS _SLEEP_PRでのスリープ時間など、バックグラウンド機能が関係する時間間隔もすべて10ms単位になります
* `-d`: バックグラウンドタスクの切り替え処理にTimer-Dを使用します
  * Human68k本来のバックグラウンドタスク機能と同等の動作になりますが、IOCS _TIMERDSTが使用できなくなります
* `-x` / `+x` : COPYキーの機能を無効化/有効化します
  * `-x`でCOPYキーを押したときの機能を完全に無効化します。trap #12が何も実行しないようになります
  * `+x`でCOPYキーが動作するようになります (デフォルト)
* `-n` / `+n` : COPYキー処理ベクタ(trap #12)の処理を変更しない/変更する
  * `-n`でtrap #12実行時の処理を変更しません。COPYキーを他の目的に使用する常駐プログラムと併用する場合などに指定してください
  * `+n`でtrap #12実行時の処理を変更して「CPUがバックグラウンドタスクを実行している瞬間にCOPYキーを押して 中止\<A\> を選択するとクラッシュすることがある」という不具合を回避します (デフォルト)
* `<スレッド数>`: 最大スレッド数
  * DOS _OPEN_PRで生成できるスレッド数の最大値を指定します (2～32)
  * 指定を省略すると32になります
  * 常駐後に設定を変更することも可能ですが、常駐時に設定した値より大きくすることはできません
* `<レベル>`: メインスレッドの優先度レベル
  * 他のバックグラウンドタスクに対するメインスレッドの優先度を指定します (2～255)
  * 値が小さいほど優先度が高くなります
  * 指定を省略すると 10 になります
* `<タイムスライス>`: タイムスライスの値
  * 複数タスクが並行して動作しているときのタスク切り替えの時間間隔を指定します (ms単位で 1～100)
  * Timer-C使用時は10ms単位に切り上げられます
  * 指定を省略すると 10ms になります


## INTERRUPTスイッチやCOPYキーによるプロセスの強制終了について

Human68kの動作中にOSやシステムのエラーで白帯が表示された場合に 中止\<A\> を選択すると、その時動いているプロセスが強制終了されてプロセスを起動したシェル等に戻ります。
これはIOCS _ABORTJOBの処理で、Human68kが実行中のプロセスの終了処理を行い、そのプロセスに設定されているエラー時戻りアドレス(通常はプロセスを起動したシェル内に設定されている)にジャンプすることで実現しています。

CPUがバックグラウンドタスクを実行している最中にこのIOCS _ABORTJOB呼び出しが行われると、強制終了されることを想定していないタスクが中断されてシェルに戻ろうとするため、システムがクラッシュします。
これを防ぐためにバックグラウンドタスクは動作中に白帯を出すようなエラーを起こさないようにする必要があるのですが、ユーザからの入力による強制終了が可能なINTERRUPTスイッチについてはアプリ側での制御ができません。バックグラウンドタスク使用時にINTERRUPTスイッチが効かないようになっていたのはこのためと思われます。

INTERRUPTスイッチと同様にユーザからの強制終了処理が可能な操作にCOPYキーがあります。プリンタがオフラインの時にCOPYキーを押すと「プリンタがつながっていません」エラーが表示されますが、ここで 中止\<A\> を選択するとINTERRUPTスイッチと同様に実行中のプロセスが強制終了されます。
この時ちょうどCPUがバックグラウンドタスクを実行中だとシステムがクラッシュするのですが、この問題については見落としたのか解決しきれなかったのか、Human68kでは放置されたままになっています。

bgenabler にはこれらの強制終了時のクラッシュを回避する処理が入っています。INTERRUPTスイッチやCOPYキーが押された際にバックグラウンドタスクが実行中だった場合はこれらの処理を一旦保留し、タスクがメインスレッドに切り替わった際に保留していた処理を実行するようにしています。
これにより、INTERRUPTスイッチやCOPYキーによる強制終了は必ずその時動作していたメインスレッドに対して行われるようになります。


## X68030 FPU命令使用時のエラー$000d対策について

Human68kには、FPUを搭載したX68030でバックグラウンドタスク機能利用時にユーザモードでFPU命令を実行すると
[エラー $000D が発生する](https://stdkmd.net/bugsx68k/#human_000d)ことがあるという不具合があります。

bgenabler は常駐時にHuman68kにこの問題を修正するためのパッチ当てを行います。
コプロセッサ接続のFPUを持たないX68000等の環境ではこの問題は発生しませんが、ごくわずかですがタスク切り替え時のパフォーマンスが向上するため、CPUの種別やFPUの有無に関係なく常にこのパッチが適用されるようになっています。

パッチ当てはパッチ未適用の状態で bgenabler が常駐する際に行われ、`Human68kのタスク切り替え処理にパッチを適用しました` と表示されます。パッチは常駐解除後も有効なまま残ります。

パッチ当て後でも、Human68kはタスクごとのFPUコンテキストを持たないため、FPU命令を使用できるのはいずれか1つのタスクに限られることに注意してください。
